Rollout order + execution notes

Recommended sequence: write all red-priority posts first (13 posts, fastest compounding SEO), then amber (23 posts), then green (14 posts).

Per-post ship checklist (reuses existing infra):

  1. Add entry to lib/blogSeo.ts with title (≤60 chars), description (150-160), category + slug, FAQs.
  2. Create app/<slug>/Content.tsx (use client) using BlogPostLayout.
  3. Create app/<slug>/page.tsx server component (copy boilerplate from any existing post).
  4. Register in lib/blogPosts.ts under the right category.
  5. Sitemap auto-picks it up. No further wiring needed.

Content quality bar: 1500-2000 words, featured-snippet answer box up top, 5-8 H2s, PAA-harvested FAQ (5-7 Qs), 5 related cross-links. No AI-generated filler — if a section needs a specific number (volume, price, stat), cite the source or drop the sentence.

Total estimated search volume: ~740k/month across all 50. Even 3-5% market share would put ResumeBuildz around 22-37k incremental monthly organic visits once the posts rank.
